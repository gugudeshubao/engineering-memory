# Repo Cards Index

把每个真实代码仓库对应到一张 repo card。

推荐命名：

- `repos/<repo-name>.md`

推荐先从当前高频仓库开始，而不是一次性补全几十个。

优先级建议：

1. 当前热工作集中的 repo
2. 高频耦合 repo
3. 最容易出事故的 repo
4. 稳定且低频的 repo
